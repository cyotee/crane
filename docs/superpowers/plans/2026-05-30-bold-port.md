# Liquity Bold Port Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Port the Liquity V2 (Bold + V2-gov) contracts and their tests from https://github.com/liquity/bold into Crane so all original tests pass under Crane's `forge` toolchain.

**Architecture:** Bold core sits at `contracts/protocols/staking/liquity/v2/bold/` mirroring `bold/contracts/src/` 1:1. V2-gov sits at `contracts/protocols/staking/liquity/v2/gov/` mirroring `bold/contracts/lib/V2-gov/src/`. Non-fork tests go to `test/foundry/spec/protocols/staking/liquity/v2/{bold,gov}/`; fork tests go to `test/foundry/fork/ethereum/protocols/staking/liquity/v2/{bold,gov}/`. OpenZeppelin imports are rewritten to use Crane's port at `contracts/external/openzeppelin-contracts/`. Solady imports map to `contracts/solady/`. forge-std comes through Crane's existing remap.

**Tech Stack:** Solidity ^0.8.35, Foundry (forge), Crane framework conventions (CLAUDE.md / AGENTS.md). EVM version: cancun.

**Reference design:** `docs/superpowers/specs/2026-05-30-bold-port-design.md` (keep updated as work proceeds — section 9 progress log).

## Conventions used throughout this plan

### Pragma rewrite (every file)
- `pragma solidity 0.8.24;` → `pragma solidity ^0.8.35;`
- `pragma solidity ^0.8.0;` (or any narrower) → `pragma solidity ^0.8.35;`
- `pragma abicoder v2;` → remove (default since 0.8.0)

### Import rewrite rules
| From | To |
|---|---|
| `openzeppelin-contracts/contracts/X` | `@crane/contracts/external/openzeppelin-contracts/X` |
| `openzeppelin/contracts/X` | `@crane/contracts/external/openzeppelin-contracts/X` |
| `Solady/utils/X` | `@crane/contracts/solady/utils/X` |
| `src/X` (in Bold tests) | `@crane/contracts/protocols/staking/liquity/v2/bold/X` |
| `forge-std/X` | `forge-std/X` (no change — remap covers it) |
| Relative imports inside src/test | preserved as-is (relative paths keep working since folder structure is mirrored) |

### License header
Bold src files use `// SPDX-License-Identifier: BUSL-1.1` — preserve exactly. Tests use MIT — preserve. Add `// Ported from liquity/bold@<commit>` comment? **No** — keep diffs clean. The submodule pointer in git history serves as provenance.

### Per-file checklist (apply mechanically unless flagged "Manual review")
1. Copy source from `lib/bold/contracts/<path>` to target location
2. Apply pragma rewrite
3. Apply import rewrite rules
4. Commit (one commit per task = one logical group of files)

## Notes for execution

- This plan deliberately uses **directory-batched** tasks rather than per-file TDD because:
  - The port is mechanical (pragma + import rewrite) for ~95% of files
  - File-by-file tests would mean 265 commits and 265 test runs — wasteful
  - The compile-check at the end of each batch is the natural verification
  - Manual-review files are flagged explicitly within their batch (`BoldToken.sol`, `TroveNFT.sol`, Zapper DEX interfaces)
- Run `forge build --skip test` frequently — it's the fastest verification step.
- If a batch breaks the build, narrow down within the batch by `git stash` + commit the working subset.
