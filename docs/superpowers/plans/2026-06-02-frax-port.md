## Conventions used throughout this plan

### Pragma rewrite (every ported `.sol`)
- `pragma solidity 0.5.x;` / `0.6.x;` / `^0.6.x;` / `>=0.6.x`-style → `pragma solidity ^0.8.35;`
- `pragma solidity 0.8.x;` (any minor) → `pragma solidity ^0.8.35;`
- `pragma solidity ^0.8.x;` → `pragma solidity ^0.8.35;`
- `pragma abicoder v2;` → remove (default since 0.8.0)
- `pragma experimental ABIEncoderV2;` → remove

### Import rewrite rules

Per [[feedback-crane-imports]] — use absolute `@crane/...` paths everywhere; relative-`./` only allowed within a tightly grouped library set.

| From | To |
|---|---|
| `@openzeppelin/contracts/X` | `@crane/contracts/external/openzeppelin-contracts/X` |
| `openzeppelin-contracts/X` | `@crane/contracts/external/openzeppelin-contracts/X` |
| `@chainlink/contracts/X` | `@crane/contracts/external/chainlink/contracts/X` (port from upstream as needed) |
| `@uniswap/v2-core/contracts/X` | `@crane/contracts/external/uniswap/v2-core/X` |
| `@uniswap/v2-periphery/contracts/X` | `@crane/contracts/external/uniswap/v2-periphery/X` |
| `@uniswap/v3-core/contracts/X` | `@crane/contracts/external/uniswap/v3-core/X` |
| `@uniswap/v3-periphery/contracts/X` | `@crane/contracts/external/uniswap/v3-periphery/X` |
| `@arbitrum/nitro-contracts/src/X` | `@crane/contracts/external/arbitrum/nitro-contracts/src/X` |
| `frax-std/X` | `@crane/contracts/external/frax-std/X` |
| `hardhat/console.sol` | `forge-std/console.sol` |
| Frax internal relative (`../Common/Context.sol` etc.) | Rewrite to absolute `@crane/contracts/protocols/tokens/stable/frax/Common/Context.sol` for any cross-subdir reference; relative `./Sibling.sol` allowed within the same subdir. |
| Test imports of `../../contracts/...` | `@crane/contracts/protocols/tokens/stable/frax/...` |
| `forge-std/X` | `forge-std/X` (no change — existing Crane remap covers it) |

**Tool reuse — DO NOT re-port if Crane already has it:**
- OpenZeppelin: already at `contracts/external/openzeppelin-contracts/` — use that.
- Chainlink AggregatorV3Interface etc.: Crane has `lib/chainlink-local` and `contracts/protocols/oracles/chainlink/` — verify if the upstream files Frax needs are present before re-porting; only port missing pieces under `contracts/external/chainlink/contracts/`.
- Uniswap V2/V3: Crane has `contracts/protocols/dexes/uniswap/`, `contracts/protocols/launchpads/uniswap/`, `contracts/interfaces/protocols/dexes/uniswap/`. **Inspect first.** If the exact interface/library file Frax imports is present (matching pragma compat), redirect; otherwise port the upstream verbatim into `contracts/external/uniswap/<v2-core|v2-periphery|v3-core|v3-periphery>/`.

### Per-file checklist (apply mechanically unless flagged "Manual review")
1. Copy source from `lib/frax-solidity/src/hardhat/contracts/<path>` to `contracts/protocols/tokens/stable/frax/<path>`.
2. Apply pragma rewrite.
3. Apply import rewrite rules (use absolute `@crane/...` for everything except same-subdir siblings).
4. Remove `pragma experimental ABIEncoderV2;` / `pragma abicoder v2;`.
5. If file is pre-0.8: see **Phase 4** modernization checklist below.
6. Commit (one commit per task = one logical group of files, scoped per subdir).

### Pre-0.8 modernization checklist (Phase 4)
For each file with original pragma `<0.8.0`:
- [ ] Remove `using SafeMath for uint*;` lines (0.8 has built-in checked arithmetic).
- [ ] Replace `.add(x)` / `.sub(x)` / `.mul(x)` / `.div(x)` / `.mod(x)` calls with native `+ - * / %`.
- [ ] Replace `SafeMath.add(a, b)` etc. with native operators.
- [ ] Convert `address(this).balance` patterns unchanged but verify `address payable` casts still parse.
- [ ] Replace `now` with `block.timestamp`.
- [ ] Wrap intentional-overflow blocks in `unchecked { ... }` (UniswapV2 reserve packing, TickMath, etc.).
- [ ] Replace `byte` with `bytes1`.
- [ ] Replace `tx.origin`/`msg.sender` casts to `payable(...)` where required.
- [ ] Remove unused SafeMath imports.
- [ ] Re-run `forge build` after each file to catch new errors immediately.

## Risks / open questions

1. **`Misc_AMOs/` (188 files) integrates with dozens of external protocols (Convex, Curve, KyberSwap, Morpho, Bunni, etc.).** Each adds transitive interface imports. Expect Phase 3.28 to dwarf the rest; may want to split per-protocol commits even smaller, and a follow-up plan if the scope is too large.
2. **`viaIR = false` in Crane** vs Frax's upstream `viaIR = true`. Some AMO contracts likely hit stack-too-deep. Mitigation per Crane convention: use structs to group locals. Track per-file in the design doc.
3. **`optimizer_runs = 1`** in Crane vs Frax's `100_000`. Bytecode size may exceed 24KB on large AMOs. May need per-contract `bytecode_hash` adjustments or split.
4. **Hardhat JS test fixtures heavily depend on truffle artifacts** (`artifacts.require(...)`). Each port has to reconstruct setup from scratch — significant effort multiplier on Phase 6.
5. **Chainlink import surface** is large — may need to port broader chunks than just the few files we see at the top of grep.
6. **Crane already has its own Uniswap V2/V3 integrations** under `contracts/protocols/dexes/uniswap/`. If pragma/version mismatches block reuse, we end up double-porting under `contracts/external/uniswap/`. Phase 1.1 audit must decide this explicitly.
7. **Fork tests across 10 chains** require RPC env vars at run time. Gate every fork test on `vm.envOr(...)` so absence of an RPC URL skips rather than fails.
